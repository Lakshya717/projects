How Custom-Attributes work?

console.log(document.querySelector('[ng-app]');

This is how AngularJS gets access to elements of html as soon as we add custom attributes (pre-defined in AngularJS framework) like ng-app, ng-controller or ng-non-bindable

$scope??

When we try to access $scope itself using console.log($scope) it returns a complex object construct. From where does this appear?



How does just by passing an argument $scope through the controller function creates such instance.

Note: $scope.name="Lakshya"; causes name property to also appear in the construct shown above.

This is explained by the following example;
A function code ShoppingCart() depends on value of charge which is decided by another function CardProcBank1() in this case. So in the code block of ShoppingCart() when using a "Regular" Control method, we would generate a new instance of the struct which will generate our charge.


However, our code is only valid as long as the outside dependencies comply and in the event of changing services from one to another, it might take a lot of effort and rewriting of code, making it more prone to bugs.


Thus to solve it we make the responsibility to create our required instances to a system js file which does all our work, using Inversion of Control


This is what AngularJS is in its essence.
