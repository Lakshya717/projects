Using Git Commands

  1. After changing cmd directory to work-folder, cloning a github project can be performed by: 
    git clone <project address>
  2. When new modifications are made to save those modifications in the local repository use: 
    git add .
  3. Status of the local repository can be viewed using 
    git status
  4. After all the required changes we can just say: 
    git commit -m "Some message"
  5. And to publish those changes on the online version of the repository use: git push

Browser-sync

  1. Initiating browser-sync feature: 
    browser-sync start --server --directory --files "**/*"
    • The --directory mode enables to see the structure of the directory in the webpage, which will open by default.
    • --files will watch for changes in the specified files in the argument which is here provided as "**/*" which means in current dir(**) watch all files(*).
  2. Note:
    • The scripts might not be able to run so review and change the ExecutionPolicy. Open Poweshell in Administrator mode and navigate to work-folder. There review the current policy by: 
      Get-ExecutionPolicy
    • And change it by using: 
      Set-ExecutionPolicy RemoteSigned
      (RemoteSigned enables only local scripts which is preffered.)

Getting Started with Angular JS

  1. Before we dive deep into this framework we actually need to get the AngularJS library from their website https://code.angularjs.org/.

    From there select suitable version and download specific files. We only reguire angular.min.js which is a minified version of angular.js (All semantics and whitespaces minimized)

    Another way is to use nodeJS package manager npm where on cmd line type: 
    npm install angular@1.8.2
    Adding a -g argument will install the library globally thus not preferred :)
  2. Now we have angular.min.js, along with this we need a JS file app.js which will act as our view . Therefore to link both of these files in html use (in order) in <head> tag; 
    <script src="angular.min.js"></script>
<script src="app.js"></script>

Configuring app.js

  1. Starting the script inside IIFE (Immediately Invoked Function Expression)
    2. ()();
    The sole purpose of this quotation is to avoid variables that bleed out in the global scope, hence everything inside this function is not valid outside(an isolated situation) 
    (function() {
    
    //commands

})();
  2. Now we will define our module property of angular using: 
    (function() {
    angular.module("myFirstApp", []);
})();
    angular.module()takes two arguments:
    • First one is the name of the app to be given "myFirstApp"
    • the second is an array of dependencies defined within [] (Here there are none)

    After defining it in app.js go to html file and hook it up in the outermost tags with a special attribute ng-app="myFirstApp" 
    <!DOCTYPE html>
<html ng-app="myFirstApp">
    <head>
        <title>Angular JS Notes
        <script src="angular.min.js">
        <script src="app.js">
    </head>
</html>
    Thus Angular JS now has access over the complete html file

  3. Another construct of angular which is responsible for smaller sections of the html file is given as: .controller property of module is specified in app.js as: 
    (function () {
    angular
    .module('myFirstApp',[])
    .controller('myFirstController',function() {
        
    });
}
)();
  4. After defining the basic structure of the controller, in html link it to any part of the tag to get its full control. 
    <div ng-controller="myFirstController">
    {{}}
</div>
    {{ }} are evaluated as AngularJS expressions

    To explicitly display them inside a html content where ng-app is present we can use : <span ng-non-bindable>{{ }}</span>
  5. Now pass a variable $scope through the function in .controller property of the .module 
    (function () {
    'use strict';

    angular.module('myFirstApp',[])

    .controller('myFirstController',function($scope) {
        $scope.name = "Lakshya"; 
    });
}
)();
    Any property defined for $scope is accessible through the html file.

    Try changing it yourself: => {{name}}

    This is how we share data between view-model and controller and our view (angular_js.html)

Name Calculator

----------------------------------------------------------
=> ASCII sum: {{input_stream_sum}}
----------------------------------------------------------
(see the working behind this functionality)

Firstly I link the data in input-box with AngularJS using ng-model attribute. Then ng-keyup acts as my eventListener and initiates the function displayNumeric();. The function and its related variables are defined as follows inside the controller property in app.js: 
(function() {
    'use strict';

    angular
    .module("myFirstApp", [])
    .controller("myFirstController",($scope) => {

        /*Some commands....*/

        $scope.input_stream = "";
        $scope.input_stream_sum = 0;

        $scope.displayNumeric = () => {
            let string = $scope.input_stream;
            let sum = 0;

            for (let i=0;i<string.length;++i)
                sum+=string.charCodeAt(i);

            $scope.input_stream_sum = sum;
        }
    });
})();
